Sample Express Application 002 : HTTPS Server

If you're reading this, it means you were able to start the http and/or https server(s). I've been running these examples on the Nodify.US Node.JS Development Kickstarter Appliance which uses a fixed range of local IP addresses. If you're doing the same, you can click on either of the two links to see this page served via HTTP or HTTPS.

  • HTTP: http://helium.l.nodify.us:10101/
  • HTTPS: http://helium.l.nodify.us:10102/

If you're running this example on a different host, you'll likely have to modify the URLs in the links above.

Also, if you click on the HTTPS link above, your browser should give you a dire warning that you're about to click on a site whose URL doesn't match the name in the certificate. This is because the certificate we distribute with this package is completely insecure. Please don't try to use it on a production system (or any system with sensitve data.)

The easiest way to create your own key and certificate is to use the OpenSSL command line utility to generate the appropriate key and CSR (Certificate Signature Request) files. You then pick a Certifying Authority to sign the public key associated with the private key you generated and follow their process to retrieve a server certificate.
