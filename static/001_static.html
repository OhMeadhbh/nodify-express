Sample Express Application 001 : Static Files

Welcome to the resource page for example application 001. You are currently viewing a HTML page served up using the static middleware from connect.js. (Express.js is layered on top of a previous package called "connect.js")

Looking at the Logs

Now that you've requested this page, you may want to visit the log file for the 001_static.js application. You should see a line that looks like this: 

192.168.56.1 - - [Wed, 15 Feb 2012 00:41:52 GMT] "GET /001_static.html HTTP/1.1" 200 833 "http://helium.l.nodify.us:10100/" "blah... blah... blah..."

If you click on a link to a resource that doesn't exist, you should see something like this in the log: 

192.168.56.1 - - [Wed, 15 Feb 2012 00:44:39 GMT] "GET /foo.html HTTP/1.1" 404 - "-" "blah... blah... blah..."

The main difference here is the 404 return code. This is just to show you that node + express can be setup to behave very similarly to apache. Visit the logger middleware page for more info.
